# General Long-Running Work Handoff Profile

Use this profile when the handoff is not primarily a code repository.

## Required Read-Only Checks

Capture the current state of the durable objects that matter for continuation:

- current working directory or workspace,
- user-stated long-term objective,
- current small task,
- important files, documents, pages, systems, accounts, or resources,
- non-sensitive entry points or identifiers,
- status of relevant external systems if they can be checked safely,
- access indexes and permission gaps,
- decisions made,
- failed attempts,
- next safe action.

Do not rely on old chat memory for facts that can change.

## Required Handoff Content

Include:

- durable objective and current mission,
- verified current facts,
- unverified or possibly stale facts,
- expired facts that should not be reused,
- resources and where to inspect them safely,
- access and secret decision summary,
- blockers,
- next actions in execution order,
- risk and rollback guidance if relevant.
